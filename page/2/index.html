参数默认值

ES6 允许为函数的参数设置默认值，即直接写在参数定义的后面。

参数变量是默认声明的，所以不能用 let 或 const 再次声明。

使用参数默认值时，函数不能有同名参数。

参数默认值不是传值的，而是每次都重新计算默认值表达式的值。也就是说，参数默认值是惰性求值的。

与解构赋值默认值结合使用

参数默认值的位置

通常情况下，定义了默认值的参数，应该是函数的尾参数。如果非尾部的参数设置默认值，实际上这个参数是没法省略的。

函数的 length 属性

指定了默认值以后，函数的 length 属性，将返回没有指定默认值的参数个数。也就是说，指定了默认值后，length 属性将失真。length 属性的含义是，该函数预期传入的参数个数。某个参数指定默认值以后，预期传入的参数个数就不包括这个参数了。

如果设置了默认值的参数不是尾参数，那么 length 属性也不再计入后面的参数了。

在参数是必须的时候，可以将参数的默认值设置为一个报错的函数，如果省略了该参数就抛出一个错误。

箭头函数

箭头函数没有参数或者有多个参数时，需要用圆括号代替参数或者将多个参数包围起来，如果只有一个参数可以省略圆括号。

如果箭头函数的代码块多于一句，则需要用大括号将其括起来，并使用 return 语句返回。
箭头函数与变量解构结合使用

使用箭头函数的注意

  1. 箭头函数的 this 对象就是定义时所在的对象。
  2. 不可以当构造函数使用，即不可以使用 new 命令。
  3. 不能使用 arguments 对象，可以用 rest 参数代替。
  4. 不可以使用 yield 命名，不能用作 generator 函数。

箭头函数中没有 this，arguments，super，这些对象都是从词法作用域中获取的。因此用 call，bind，apply 等方法是无法改变箭头函数中的 this 指向的。
稳定的 this 指向，使箭头函数非常适合用来封装回调函数。

当点击一个按钮时，普通函数回调函数的 this 一般会指向 document 对象，这样就容易出现 this 对象上没有相应的方法，箭头函数解决了这个问题。

箭头函数不适合的场景

  1. 在对象中定义方法。
    在对象中定义方法，由于对象先没有单独的作用域，因此箭头函数中的 this 会指向全局作用域，同时也不方便对象方法的复用。

  2. 需要动态 this 的场景使用。

Symbol 的定义

Symbol 是 JavaScript 第六种基本类型，代表一个独一无二的值。

Symbol 的创建

由于 Symbol 是基本类型，因此不能用 new Symbol()来创建，否则会报错。
创建一个无标记的 Symbol

创建一个有标记的 Symbol，主要目的方便区分不同的 Symbol，否则打印出来都是 Symbol();

通过 Symbol.for(key)，可以创建一个类似于单例模式的 Symbol，以后每一次调用 Symbol.for(key)都是返回同一个对象。

需要注意的是，Symbol.for 为 Symbol 值登记的名字，是全局环境的，可以在不同的 iframe 或 service worker 中取到同一个值。
通过调用 Symbol.keyfor(symbol 值)，可以得到该 Symbol 值创建时对应的 key。

Symbol 作为属性值

在对象的内部，使用 Symbol 值定义属性时，Symbol 值必须放在方括号之中。

Symbol 属性值的遍历

Symbol 作为属性名，该属性不会出现在 for...in、for...of循环中，也不会被Object.keys()、Object.getOwnPropertyNames()、JSON.stringify()返回。
但是，它也不是私有属性，有一个 Object.getOwnPropertySymbols 方法，可以获取指定对象的所有 Symbol 属性名。
Object.getOwnPropertySymbols 方法返回一个数组，成员是当前对象的所有用作属性名的 Symbol 值。

内置的 Symbol 值

Symbol.hasInstance

当其他对象使用 instanceof 运算符，判断是否为该对象的实例时，会调用这个方法。

Symbol.isConcatSpreadable

对象的 Symbol.isConcatSpreadable 属性等于一个布尔值，表示该对象用于 Array.prototype.concat()时，是否可以展开。

类似数组的对象正好相反，默认不展开。它的 Symbol.isConcatSpreadable 属性设为 true，才可以展开。

let-and-const

发表于 2019-10-04

let不存在变量提升，同时let还会绑定所在的代码块，即大括号包含的区域。在这个代码块内如果在let声明之前使用变量，就会报ReferenceError错误。

let 不可以重复声明变量。

let块级作用域和函数声明
在 ES6 中可以在块级作用域中声明函数，声明行为类似于let。但是在实际 ES6 浏览器中，函数声明有自己的方式。
允许在块级作用域声明函数，并且函数会提升到该块级作用域顶部，同时会提升到全局作用域或者函数作用域顶部
。

由于不同环境下造成的差异较大，因此在块级作用域中应该避免声明函数，或者应该使用函数表达式。

const声明变量的行为与let基本一致，const表示声明一个常量，不同多次赋值，因此一定要在声明的时候同时初始化。

当const声明的常量是一个对象时，实际上不变性指的只是对象的地址值保持不变。因此堆中的对象的属性值发生改变时，const是不会报错的。

但是当常量的地址值改变了，此时常量的值就真正改变了，就会报错。

要包持对象的不变性，可以对对象进行封装或者冻结。

ES6 中一共有六种声明变量的方法：var，function，let，const，class，import

顶层对象的属性
var，function声明的全局变量依旧是顶级对象的属性，let，const，class命令声明的全局变量不属于顶级对象。

给一个对象设置属性时，当一个对象的原型链上已经存在该属性，此时会发生三种情况

    • 对象常量
      通过将configurable和writable属性设置为 false，可以为对象创建一个常量属性。

      禁止一个对象添加新属性并且保留已有属性，可以使用Object.

      Object.seal()方法先禁止对象扩展，并把对象所有属性标记为configurable为false，添加后对象不仅不能添加新的属性，也不能重新配置和删除已有的属性。

    • 冻结
      Object.freeze(),创建一个冻结对象，所有属性的writable都被设置被false,这个方法是可以应用在对象上最高级别的不变性。

  • Getter和Setter
    当一个对象定义getter或者setter属性，JavaScript 会忽略对象的value和writable特性，而通过set和get特性去获取或者设置对象的值。

    判断对象是否存在某一个属性，可以用in，hasOwnProperty

    in会查找整个对象自身及其原型链的属性，hasOwnProperty只会查找对象本身的属性，这两个判断的是属性的可访问性。

    可枚举性意味着可以被遍历，遍历指的是for...in和Object.keys()等具有循环的方法，Object.keys()可以获取对象本身所有可枚举属性，但是没有内置函数可以得到对象及其原型链上所有可枚举属性的列表。判断对象本身属性是否可以枚举可以使用 propertyIsEnumerable()，getOwnPropertyNames()可以获取对象自身所有可访问属性，它不是一个遍历方法。

  • this 是什么？

    在 JavaScript 的函数中,this 总是指向一些对象，这些对象可以是全局的对象 window，也可以是某一个用户自定义的对象，在严格模式下也可以是 undefined。this 并非指向函数声明时所在的环境，而在运行时基于函数的执行环境动态绑定的。

    上述代码第一个 foo 函数执行环境为 window，第二个 foo 函数执行环境为 obj，因此 this 的指向是动态绑定的。

  • 为什么要用 this？

    通过使用 this，当函数被不同的对象调用时，可以避免传入对象作为参数，而是直接通过改变 this 的指向即可。

    通过使用 this，可以使我们的代码更加优雅，提高代码的可复用性。

  • this 的绑定规则

    • 默认绑定
      在非严格模式下，普通函数调用时 this 默认绑定的全局对象，在严格模式下，普通函数的 this 绑定的是 undefined。

    • 隐式绑定

      当函数调用位置有上下下文时，或者说是否被某一个对象调用时，this 会绑定到这个上下文对象中。

      此处 obj 调用了 foo，因此 foo 中的 this 绑定到了 obj 对象上。

      • 链式调用

        链式调用时，函数的 this 始终绑定到最后一层调用上，即 obj1。

    • 显示绑定

      显示绑定即利用 apply,call 等方法将函数的 this 绑定到指定的对象。

      • 硬绑定

      clickHandler 一个 this 始终与全局对象绑定的函数，clickHanler 再绑定其他对象也是没有用的。

  • 箭头函数的 this
    箭头函数是根据外层的作用域，如函数作用域和全局作用域来决定 this 的，因此不遵循上述的 4 中规则。
