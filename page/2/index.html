promise

发表于 2019-10-11

基本用法

promise是一个构造函数，参数是一个函数，且这个函数接收两个函数参数，一个是 resolve 函数，一个是 reject 函数。resolve 函数负责将 promise 的状态设置为 fullfill 即完成状态，而 reject 负责将 promise 状态设置为 reject 的状态，即失败状态。promise 的实例对象有一个 then 方法和 catch 方法，在 fullfill 状态调用 then 回调函数，在 reject 状态调用 catch 函数。

如果调用 resolve 函数和 reject 函数时带有参数，那么它们的参数会被传递给回调函数。reject 函数的参数通常是 Error 对象的实例，表示抛出的错误；resolve 函数的参数除了正常的值以外，还可能是另一个 Promise 实例，比如像下面这样。

p2 的状态由 p1 决定，因此 p2 的 then，catch 都是针对 p1 的。

Promise.prototype.then

promise原型对象上的 then 方法是指定状态之后执行的回调函数，then 方法两个函数参数，第一个是 resolve 对应的回调函数，第二个是 reject 对应的回调函数。then 方法返回一个新的 promise 实例，因此可以链式调用。

在链式调用中返回 promise

Promise.prototype.catch

catch 方法用来捕获 promise 的错误，即 reject 状态。当 promise 中的抛出一个错误时，也会触发 catch。

catch 相当于 then(null,catch)的缩写
当有多个 promise 链式调用时，应该将 catch 放到最后，捕获所有 promise 的错误。

如果没有使用 catch 方法指定错误处理的回调函数，Promise 对象抛出的错误不会传递到外层代码，即不会有任何反应。

Promise.prototype.finally

finally用于指定不管promise最后的状态是什么都会执行的操作，finally(callback)回调函数不接收任何参数，可以用()=>dosomething 代替，也就是说 finally 的执行与否与状态无关。

Promise.all

all方法接收一个 promise 实例组成的数组，或者具有迭代器的结构，返回一个新的promise实例。当数组中所有的promise为 resolve 后，返回的promise执行的状态为resolve返回所有resolve组成的数组，当数组中有一个状态为reject，返回的promise的状态为reject，且为第一个返回的reject。

有一个reject就触发catch。

如果数组中的promise中有catch回调函数，那么该promise返回的是catch返回的promise，而且其状态为resolve，返回为空值。

node 环境下数组中的promise和all函数返回的promise都会触发catch函数。

如果指定时间内没有获得结果，就将 Promise 的状态变为 reject，否则变为 resolve。

Promise.allSettled()

Promise.allSettled()方法接受一组 Promise 实例作为参数，包装成一个新的 Promise 实例。只有等到所有这些参数实例都返回结果，不管是 fulfilled 还是 rejected，包装实例才会结束。

不关心异步操作的结果，只关心这些操作有没有结束。这时，Promise.allSettled()方法就很有用。

Promise.any()

接收一组promise实例作为参数，返回一个包装promise函数，只有这一组promise有一个状态为 fullfilled，那么包装的promise的状态就为该fullfilled；要所有的promise的状态为reject,包装的promise状态才为reject，传入回调函数的值为所有reject返回值的数组。

Promise.resolve()

将传入的参数转化为promise对象，有四种情况：

  1. 传入Promise对象，不做任何处理，返回参数。
  2. 传入thenable数据结构，转化为Promise对象后马上执行参数的then方法。
  1. 参数不是具有 then 方法的对象，或根本就不是对象，返回状态为fullfilled的promise对象，并将参数作为返回值。
  1. 不传参数，返回状态为fullfilled的promise对象。立即 resolve()的 Promise 对象，是在本轮“事件循环”（event loop）的结束时执行，而不是在下一轮“事件循环”的开始时。

proxy

发表于 2019-10-10

proxy 是什么？

proxy 是一个构造函数，用来构造指定对象的代理对象。

基本用法

Proxy 函数接受两个参数，第一个是要代理的对象，第二个是配置对象，其中包括要被代理的操作，比如 get 操作（获取属性），set 操作（设置属性），apply（函数被调用）等操作。这些操作执行时，就会被拦截执行另外的操作。

可以将 proxy 设置为目标对象的一个属性，这样就可以通过对象属性访问代理对象。

proxy 也可以作为其他对象的原型

proxy 拦截方法

get()

接收三个参数，第一个为目标对象，第二个为属性，第三个为 proxy 实例
使用 get 拦截，实现数组读取负数的索引。

利用 get 参数可以实现函数的链式调用
思路：先将访问的属性如 double，pow 等存放在一个数组中，且每次访问都返回 proxy 对象。当调用 get 方法时，一次性返回或者如果属性是函数则执行所有函数。

set()

set 在代理对象设置属性时调用，接收四个参数，第一个是目标对象，第二个是属性名，第三个是属性值，第四个是 proxy 实例。

对年龄的设置限制条件，超过 200 则报错。
set 和 get 还可以结合使用用来设置私有属性

Apply

apply 拦截函数的调用，call，apply 等操作。

has

拦截 HasProperty，即判断对象是否有某个属性时，如 in 操作。可以利用这个方法隐藏私有属性。

has 拦截只对 in 运算符生效，对 for…in 循环不生效，导致不符合要求的属性没有被 for…in 循环所排除。

deleteProperty

deleteProperty(target, propKey)：拦截 delete proxy[propKey]的操作，返回一个布尔值。

ownkeys

ownKeys(target)：拦截 Object.getOwnPropertyNames(proxy)、Object.getOwnPropertySymbols(proxy)、Object.keys(proxy)、for…in 循环，返回一个数组。该方法返回目标对象所有自身的属性的属性名，而 Object.keys()的返回结果仅包括目标对象自身的可遍历属性。

getOwnPropertyDescriptor(target, propKey)

拦截 Object.getOwnPropertyDescriptor(proxy, propKey)，返回属性的描述对象。

defineProperty(target, propKey, propDesc)

拦截 Object.defineProperty(proxy, propKey, propDesc）、Object.defineProperties(proxy, propDescs)，返回一个布尔值。

preventExtensions

拦截 Object.preventExtensions(proxy)，返回一个布尔值。

getPrototypeOf(target)

拦截 Object.getPrototypeOf(proxy)，返回一个对象。

isExtensible(target)

拦截 Object.isExtensible(proxy)，返回一个布尔值。

setPrototypeOf(target, proto)

拦截 Object.setPrototypeOf(proxy, proto)，返回一个布尔值。如果目标对象是函数，那么还有两种额外操作可以拦截。

construct(target, args)

拦截 Proxy 实例作为构造函数调用的操作，比如 new proxy(…args)。

this 问题

当 proxy 对象调用目标对象的函数时，实际上 this 是指向 proxy 实例对象的。因此有时会造成无法代理目标对象。

有些原生对象的内部属性，只有通过正确的 this 才能拿到，所以 Proxy 也无法代理这些原生对象的属性。

解决办法是可以返回一个 this 与目标对象强绑定的函数

web 服务器

服务器有一个接口对象，不同的数据需要写一个适配方法

利用 proxy 拦截属性，可以只使用一个接口

Set 是什么？

Set 是一种 js 一种新的数据结构，类似于数组，属于对象，特点是成员的值都是唯一的，没有重复的值。

可以用扩展运算符实现数组去重

也可以实现字符串的去重

Set 中判断值相等的方式于“===”类似，但是 NaN 与 NaN 是相等的。

Set 实例的方法

操作数据的方法
增加Set.prototype.add(v)返回 set 本身，删除Set.prototype.delete(v)，判断是否有指定的值Set.prototype.has(v)，清空 set 结构Set.prototype.clear()

遍历数组的方法
Set.prototype.keys()，Set.prototype.values()，Set.prototype.entries()，Set.forEach()前三种方法都是返回遍历器对象，即只能使用手动 next 方法或者用 for…of…调用遍历器的 next 方法。

通过扩展运算符（for…of…的缩写）可以将 Set 结构转化为数组，使用数组的 filter 方法，可以实现交集，并集，差集。

WeakSet

weakSet 的性质和实例方法和 Set 结构类似，但是 WeakSet 有以下特点
成员只能是对象，且不能被遍历。weakSet 结构中对对象都是弱引用，也就是当 weakSet 中的对象的其他引用为 0，这些对象可以被回收。

Map 结构

Js 对象本质上是键值对的集合（Hash 结构），但是传统上只能用字符串当作键。如将 dom 节点作为键值则 dom 节点会被转化为字符串[object HTMLDivElement]
Map 结构可以将任何类型的数据作为键值，因为 Map 结构不会管键值到底是什么，只会找到键值所在的内存地址。

Map 的键实际上是跟内存地址绑定的，只要内存地址不一样，就视为两个键。这就解决了同名属性碰撞（clash）的问题，我们扩展别人的库的时候，如果使用对象作为键名，就不用担心自己的属性与原作者的属性同名。

Map 实例的方法
Map.prototype.set()返回 Map 实例，可以作链式操作，Map.prototype.get(k)，获取键值对应的键名，Map.prototype.has(key)，Map.prototype.delete(key)这些方法都和 Set 一致。

遍历方法包括 keys(),values(),entries(),forEach(),但是默认是可以被 for…of…遍历的，默认的遍历器对象为 entries()返回的对象。

Map 可以转化为数组进行处理后再转化为 Map 结构，也就是说可以用数组的方法对象 Map 数据进行处理

与其他结构的转换

weakMap

weakMap 的键名只能是对象，也是弱引用，不影响垃圾回收。
用途可以用 DOM 节点作为键名。

dom 节点删除，该状态就自动消失，可以防止内存泄漏

WeakMap 的另一个用处是部署私有属性。

Countdown 类的两个内部属性_counter 和_action，是实例的弱引用，所以如果删除实例，它们也就随之消失，不会造成内存泄漏。

Object.assign()

Object.assign()用于对象可枚举属性的合并，第一个参数是目标对象，其他参数都是源对象，出现相同属性时后面的对象会覆盖前面的对象。

如果参数不是对象，该方法会将其转为为对象，然后返回，但是原来的值没有变化。

如果无法转化为对象，则报错。

如果非对象参数出现在源对象的位置（即非首参数），那么处理规则有所不同。首先，这些参数都会转成对象，如果无法转成对象，就会跳过。这意味着，如果 undefined 和 null 不在首参数，就不会报错。
其他类型的值（即数值、字符串和布尔值）不在首参数，也不会报错。但是，除了字符串会以数组形式，拷贝入目标对象，其他值都不会产生效果。

属性的可枚举性和遍历

可枚举的属性即 enumerator 属性为 true 的属性，这些属性可以被 for..in..，object,keys()，Object.assign()，JSON.stringify，等方法遍历，其中 for…in…会遍历原型链的属性。引入可枚举的目的是为了让某些属性可以规避掉 for…in..操作，否则很多对象原型上的方法如 toString()都会被遍历出来。

ES6 规定，所有 Class 的原型的方法都是不可枚举的。

操作中引入继承的属性会让问题复杂化，大多数时候，我们只关心对象自身的属性。所以，尽量不要用 for…in 循环，而用 Object.keys()代替。

ES6 遍历属性的 5 种方法

  1. for…in…
    遍历对象自身和原型链的可枚举属性，不含 symbol
  2. Object.keys()
    遍历对象自身的可枚举属性，不含 symbol
  3. Object.getOwnPropertyNames()
    遍历对象自身可枚举和不可枚举属性，不包含 symbol
  4. Object.getOwnPropertySymbols
    遍历对象自身所有 Symbol 属性
  5. Reflect.ownKeys(obj)
    Reflect.ownKeys 返回一个数组，包含对象自身的所有键名，不管键名是 Symbol 或字符串，也不管是否可枚举。

copyWithin()

copyWithin(target,start,end)，将指定位置即 target 的值替换为 start 到 end 之间的值，不包括 end。

indexOf 方法有两个缺点，一是不够语义化，它的含义是找到参数值的第一个出现位置，所以要去比较是否不等于-1，表达起来不够直观。二是，它内部使用严格相等运算符（===）进行判断，这会导致对 NaN 的误判。
Array.prototype.includes 方法返回一个布尔值，表示某个数组是否包含给定的值。语义化且可以判断 NAN。

Map 结构的 has 方法，是用来查找键名的，比如 Map.prototype.has(key)、WeakMap.prototype.has(key)、Reflect.has(target, propertyKey)。
Set 结构的 has 方法，是用来查找值的，比如 Set.prototype.has(value)、WeakSet.prototype.has(value)。

flat()，flatMap()

数组的成员有时还是数组，Array.prototype.flat()用于将嵌套的数组“拉平”，变成一维的数组。该方法返回一个新数组，对原数据没有影响。

flat()默认只会“拉平”一层，如果想要“拉平”多层的嵌套数组，可以将 flat()方法的参数写成一个整数，表示想要拉平的层数，默认为 1。

如果不管有多少层嵌套，都要转成一维数组，可以用 Infinity 关键字作为参数。

flatMap()方法对原数组的每个成员执行一个函数（相当于执行 Array.prototype.map()），然后对返回值组成的数组执行 flat()方法。该方法返回一个新数组，不改变原数组。

数组的空位

由于空位的处理规则非常不统一，所以建议避免出现空位。

参数默认值

ES6 允许为函数的参数设置默认值，即直接写在参数定义的后面。

参数变量是默认声明的，所以不能用 let 或 const 再次声明。

参数默认值的位置

通常情况下，定义了默认值的参数，应该是函数的尾参数。如果非尾部的参数设置默认值，实际上这个参数是没法省略的。

函数的 length 属性

指定了默认值以后，函数的 length 属性，将返回没有指定默认值的参数个数。也就是说，指定了默认值后，length 属性将失真。length 属性的含义是，该函数预期传入的参数个数。某个参数指定默认值以后，预期传入的参数个数就不包括这个参数了。

如果设置了默认值的参数不是尾参数，那么 length 属性也不再计入后面的参数了。

使用箭头函数的注意

  1. 箭头函数的 this 对象就是定义时所在的对象。
  2. 不可以当构造函数使用，即不可以使用 new 命令。
  3. 不能使用 arguments 对象，可以用 rest 参数代替。
  4. 不可以使用 yield 命名，不能用作 generator 函数。

箭头函数中没有 this，arguments，super，这些对象都是从词法作用域中获取的。因此用 call，bind，apply 等方法是无法改变箭头函数中的 this 指向的。
稳定的 this 指向，使箭头函数非常适合用来封装回调函数。

当点击一个按钮时，普通函数回调函数的 this 一般会指向 document 对象，这样就容易出现 this 对象上没有相应的方法，箭头函数解决了这个问题。

箭头函数不适合的场景

  1. 在对象中定义方法。
    在对象中定义方法，由于对象先没有单独的作用域，因此箭头函数中的 this 会指向全局作用域，同时也不方便对象方法的复用。

  2. 需要动态 this 的场景使用。

Symbol 的定义

Symbol 是 JavaScript 第六种基本类型，代表一个独一无二的值。

Symbol 的创建

由于 Symbol 是基本类型，因此不能用 new Symbol()来创建，否则会报错。
创建一个无标记的 Symbol

创建一个有标记的 Symbol，主要目的方便区分不同的 Symbol，否则打印出来都是 Symbol();

通过 Symbol.for(key)，可以创建一个类似于单例模式的 Symbol，以后每一次调用 Symbol.for(key)都是返回同一个对象。

需要注意的是，Symbol.for 为 Symbol 值登记的名字，是全局环境的，可以在不同的 iframe 或 service worker 中取到同一个值。
通过调用 Symbol.keyfor(symbol 值)，可以得到该 Symbol 值创建时对应的 key。

Symbol 作为属性值

在对象的内部，使用 Symbol 值定义属性时，Symbol 值必须放在方括号之中。

Symbol 属性值的遍历

Symbol 作为属性名，该属性不会出现在 for...in、for...of循环中，也不会被Object.keys()、Object.getOwnPropertyNames()、JSON.stringify()返回。
但是，它也不是私有属性，有一个 Object.getOwnPropertySymbols 方法，可以获取指定对象的所有 Symbol 属性名。
Object.getOwnPropertySymbols 方法返回一个数组，成员是当前对象的所有用作属性名的 Symbol 值。
