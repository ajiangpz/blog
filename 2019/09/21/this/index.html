  • this 是什么？

    在 JavaScript 的函数中,this 总是指向一些对象，这些对象可以是全局的对象 window，也可以是某一个用户自定义的对象，在严格模式下也可以是 undefined。this 并非指向函数声明时所在的环境，而在运行时基于函数的执行环境动态绑定的。

    上述代码第一个 foo 函数执行环境为 window，第二个 foo 函数执行环境为 obj，因此 this 的指向是动态绑定的。

  • 为什么要用 this？

    通过使用 this，当函数被不同的对象调用时，可以避免传入对象作为参数，而是直接通过改变 this 的指向即可。

    通过使用 this，可以使我们的代码更加优雅，提高代码的可复用性。

  • this 的绑定规则

    • 默认绑定
      在非严格模式下，普通函数调用时 this 默认绑定的全局对象，在严格模式下，普通函数的 this 绑定的是 undefined。

    • 隐式绑定

      当函数调用位置有上下下文时，或者说是否被某一个对象调用时，this 会绑定到这个上下文对象中。

      此处 obj 调用了 foo，因此 foo 中的 this 绑定到了 obj 对象上。

      • 链式调用

        链式调用时，函数的 this 始终绑定到最后一层调用上，即 obj1。

      显示绑定即利用 apply,call 等方法将函数的 this 绑定到指定的对象。

      • 硬绑定

      clickHandler 一个 this 始终与全局对象绑定的函数，clickHanler 再绑定其他对象也是没有用的。

    • new 绑定
      通过 new 调用函数时，如果函数中没有返回对象，那么函数 this 绑定的就是 new 生成的实例。

      注意当构造函数中返回了一个对象，此时 this 绑定的就不是返回的对象了。

  • 箭头函数的 this
    箭头函数是根据外层的作用域，如函数作用域和全局作用域来决定 this 的，因此不遵循上述的 4 中规则。

    bar(箭头函数)的 this 指向的是 window,因为 foo 调用时其 this 就是全局变量，baz 的 this 指向 obj，同理。箭头函数的绑定无法被修改(就算是 new 也不行！)

  • 如何使用 this

    • 只使用词法作用域，完全抛弃错误 this 风格的代码
    • 完全采用 this 风格的代码，必要时使用 bind，尽量避免使用 self=this 和箭头函数。
