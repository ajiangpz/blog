错误处理

async 函数返回的是 promise 对象，内部函数的返回值会被 then 函数接收到。如果抛出错误，会被 catch 接收到

async 返回的 promise 必须在函数内所有 await 后面的 promise 对象执行完才会发生状态改变，除非遇到 return 或者抛出错误。

await

await 后面接一个 promise 对象，如果不是 promise 对象，则直接返回对应的值。thenable 对象在此处也视为 promise 对象，执行 then 方法之后，通过 resolve 返回值。

如果想要 await 抛出错误后 async 函数不中断执行，可以在 await 后的 promise 对象加一个 catch 函数。

利用 try…catch 实现多次尝试异步操作

使用注意点

  1. 最好把 await 命令放在 try…catch…代码块中，
  2. 多个 await 的异步操作，如果不是继发关系，最好让它们同时触发。
  1. await 命令只能用在 async 函数之中，如果用在普通函数，就会报错。forEach(function(){})函数参数中如果不是 async 函数也不可以用 await，如果函数参数是 async 的函数，那么这些异步请求会并发执行。

因此，在循环结构中使用 async，应该使用 for()循环。

  1. async 函数可以保留运行堆栈。
