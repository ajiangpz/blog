set-and-map

发表于 2019-10-08

Set 是什么？

Set 是一种 js 一种新的数据结构，类似于数组，属于对象，特点是成员的值都是唯一的，没有重复的值。

可以用扩展运算符实现数组去重

也可以实现字符串的去重

Set 中判断值相等的方式于“===”类似，但是 NaN 与 NaN 是相等的。

Set 实例的方法

操作数据的方法
增加Set.prototype.add(v)返回 set 本身，删除Set.prototype.delete(v)，判断是否有指定的值Set.prototype.has(v)，清空 set 结构Set.prototype.clear()

遍历数组的方法
Set.prototype.keys()，Set.prototype.values()，Set.prototype.entries()，Set.forEach()前三种方法都是返回遍历器对象，即只能使用手动 next 方法或者用 for…of…调用遍历器的 next 方法。

通过扩展运算符（for…of…的缩写）可以将 Set 结构转化为数组，使用数组的 filter 方法，可以实现交集，并集，差集。

WeakSet

weakSet 的性质和实例方法和 Set 结构类似，但是 WeakSet 有以下特点
成员只能是对象，且不能被遍历。weakSet 结构中对对象都是弱引用，也就是当 weakSet 中的对象的其他引用为 0，这些对象可以被回收。

Map 结构

Js 对象本质上是键值对的集合（Hash 结构），但是传统上只能用字符串当作键。如将 dom 节点作为键值则 dom 节点会被转化为字符串[object HTMLDivElement]
Map 结构可以将任何类型的数据作为键值，因为 Map 结构不会管键值到底是什么，只会找到键值所在的内存地址。

Map 的键实际上是跟内存地址绑定的，只要内存地址不一样，就视为两个键。这就解决了同名属性碰撞（clash）的问题，我们扩展别人的库的时候，如果使用对象作为键名，就不用担心自己的属性与原作者的属性同名。

Map 实例的方法
Map.prototype.set()返回 Map 实例，可以作链式操作，Map.prototype.get(k)，获取键值对应的键名，Map.prototype.has(key)，Map.prototype.delete(key)这些方法都和 Set 一致。

遍历方法包括 keys(),values(),entries(),forEach(),但是默认是可以被 for…of…遍历的，默认的遍历器对象为 entries()返回的对象。

Map 可以转化为数组进行处理后再转化为 Map 结构，也就是说可以用数组的方法对象 Map 数据进行处理

与其他结构的转换

weakMap

weakMap 的键名只能是对象，也是弱引用，不影响垃圾回收。
用途可以用 DOM 节点作为键名。

dom 节点删除，该状态就自动消失，可以防止内存泄漏

WeakMap 的另一个用处是部署私有属性。

Countdown 类的两个内部属性_counter 和_action，是实例的弱引用，所以如果删除实例，它们也就随之消失，不会造成内存泄漏。
