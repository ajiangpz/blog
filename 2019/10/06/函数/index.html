参数默认值

ES6 允许为函数的参数设置默认值，即直接写在参数定义的后面。

参数变量是默认声明的，所以不能用 let 或 const 再次声明。

使用参数默认值时，函数不能有同名参数。

参数默认值不是传值的，而是每次都重新计算默认值表达式的值。也就是说，参数默认值是惰性求值的。

参数默认值的位置

通常情况下，定义了默认值的参数，应该是函数的尾参数。如果非尾部的参数设置默认值，实际上这个参数是没法省略的。

函数的 length 属性

指定了默认值以后，函数的 length 属性，将返回没有指定默认值的参数个数。也就是说，指定了默认值后，length 属性将失真。length 属性的含义是，该函数预期传入的参数个数。某个参数指定默认值以后，预期传入的参数个数就不包括这个参数了。

如果设置了默认值的参数不是尾参数，那么 length 属性也不再计入后面的参数了。

默认值的应用

在参数是必须的时候，可以将参数的默认值设置为一个报错的函数，如果省略了该参数就抛出一个错误。

rest 参数

用于获取函数的多余参数，这样就不需要使用 arguments 对象了。rest 参数搭配的变量是一个数组，该变量将多余的参数放入数组中。

相比 arguments 参数这种类数组对象，rest 参数是一个真正的数组，不需要转化。

name 属性

ES6 的函数可以的 name 属性可以返回函数的名称。

Function 构造函数返回的函数实例，name 属性的值为 anonymous。bind 返回的函数，name 属性值会加上 bound 前缀。

箭头函数

箭头函数没有参数或者有多个参数时，需要用圆括号代替参数或者将多个参数包围起来，如果只有一个参数可以省略圆括号。

如果箭头函数的代码块多于一句，则需要用大括号将其括起来，并使用 return 语句返回。
箭头函数与变量解构结合使用

使用箭头函数的注意

  1. 箭头函数的 this 对象就是定义时所在的对象。
  2. 不可以当构造函数使用，即不可以使用 new 命令。
  3. 不能使用 arguments 对象，可以用 rest 参数代替。
  4. 不可以使用 yield 命名，不能用作 generator 函数。

箭头函数中没有 this，arguments，super，这些对象都是从词法作用域中获取的。因此用 call，bind，apply 等方法是无法改变箭头函数中的 this 指向的。
稳定的 this 指向，使箭头函数非常适合用来封装回调函数。

当点击一个按钮时，普通函数回调函数的 this 一般会指向 document 对象，这样就容易出现 this 对象上没有相应的方法，箭头函数解决了这个问题。

箭头函数不适合的场景

  1. 在对象中定义方法。
    在对象中定义方法，由于对象先没有单独的作用域，因此箭头函数中的 this 会指向全局作用域，同时也不方便对象方法的复用。
