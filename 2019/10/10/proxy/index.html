proxy

发表于 2019-10-10

proxy 是什么？

proxy 是一个构造函数，用来构造指定对象的代理对象。

基本用法

Proxy 函数接受两个参数，第一个是要代理的对象，第二个是配置对象，其中包括要被代理的操作，比如 get 操作（获取属性），set 操作（设置属性），apply（函数被调用）等操作。这些操作执行时，就会被拦截执行另外的操作。

可以将 proxy 设置为目标对象的一个属性，这样就可以通过对象属性访问代理对象。

proxy 也可以作为其他对象的原型

proxy 拦截方法

get()

接收三个参数，第一个为目标对象，第二个为属性，第三个为 proxy 实例
使用 get 拦截，实现数组读取负数的索引。

利用 get 参数可以实现函数的链式调用
思路：先将访问的属性如 double，pow 等存放在一个数组中，且每次访问都返回 proxy 对象。当调用 get 方法时，一次性返回或者如果属性是函数则执行所有函数。

set()

set 在代理对象设置属性时调用，接收四个参数，第一个是目标对象，第二个是属性名，第三个是属性值，第四个是 proxy 实例。

对年龄的设置限制条件，超过 200 则报错。
set 和 get 还可以结合使用用来设置私有属性

Apply

apply 拦截函数的调用，call，apply 等操作。

has

拦截 HasProperty，即判断对象是否有某个属性时，如 in 操作。可以利用这个方法隐藏私有属性。

has 拦截只对 in 运算符生效，对 for…in 循环不生效，导致不符合要求的属性没有被 for…in 循环所排除。

deleteProperty

deleteProperty(target, propKey)：拦截 delete proxy[propKey]的操作，返回一个布尔值。

ownkeys

ownKeys(target)：拦截 Object.getOwnPropertyNames(proxy)、Object.getOwnPropertySymbols(proxy)、Object.keys(proxy)、for…in 循环，返回一个数组。该方法返回目标对象所有自身的属性的属性名，而 Object.keys()的返回结果仅包括目标对象自身的可遍历属性。

getOwnPropertyDescriptor(target, propKey)

拦截 Object.getOwnPropertyDescriptor(proxy, propKey)，返回属性的描述对象。

defineProperty(target, propKey, propDesc)

拦截 Object.defineProperty(proxy, propKey, propDesc）、Object.defineProperties(proxy, propDescs)，返回一个布尔值。

preventExtensions

拦截 Object.preventExtensions(proxy)，返回一个布尔值。

getPrototypeOf(target)

拦截 Object.getPrototypeOf(proxy)，返回一个对象。

isExtensible(target)

拦截 Object.isExtensible(proxy)，返回一个布尔值。

setPrototypeOf(target, proto)

拦截 Object.setPrototypeOf(proxy, proto)，返回一个布尔值。如果目标对象是函数，那么还有两种额外操作可以拦截。

construct(target, args)

拦截 Proxy 实例作为构造函数调用的操作，比如 new proxy(…args)。

this 问题

当 proxy 对象调用目标对象的函数时，实际上 this 是指向 proxy 实例对象的。因此有时会造成无法代理目标对象。

有些原生对象的内部属性，只有通过正确的 this 才能拿到，所以 Proxy 也无法代理这些原生对象的属性。

解决办法是可以返回一个 this 与目标对象强绑定的函数

web 服务器

服务器有一个接口对象，不同的数据需要写一个适配方法

利用 proxy 拦截属性，可以只使用一个接口
