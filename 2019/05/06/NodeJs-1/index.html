NodeJs

代码组织和部署

模块路径解析规则

  • 内置模块

  • node_modules目录
    如果文件的绝对路径为/a/b/c

    会逐级向上搜索node_modules，即

    1. ‘a/b/c/node_modules/foo/bar’
    2. ‘a/b/node_modules/foo/bar’
    3. ‘a/node_modules/foo/bar’
    4. ‘node_modules/foo/bar’
  • 自定义模块搜索路径
    在环境变量中配置node_path，加载模块时就会去该目录

包

  • 有多个依赖模块，将这些模块放在一个文件夹下，定义入口模块和输出模块，就形成了一个包
  • 包的入口模块默认时index.js,因此命名时如果为index.js就可以避免将输入模块写进路径中
  • 可以通过在包的根目录下创建package.json配置输入模块

命令行程序

  • window下配置命令行的路径，这样在任何地方都可以调用该程序

  • 在同一目录创建node-echo.cmd文件，在文件中加入以下代码，表示要执行的脚本

  • 在cmd规定的位置创建node-echo.js文件，编写要执行的内容

工程目录

目录 作用
bin 命令行代码
lib APIP
doc 文档
test 测试
readme.md 项目简单介绍
package.josn 项目包配置

NPM

  • 安装第三方包
    • npm i
  • 安装命令行工具
    • npm i -g
  • 发布第三方包和命令行程序
    • npm addUser 注册npm用户
    • 编写包
      • package.json中配置必要的选项
