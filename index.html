执行栈

在调用一个函数时JavaScript引擎会将函数的变量对象入栈，当函数返回时，则将函数的变量对象出栈。如果所有的函数都执行完毕了，那么此时执行栈为空。

消息队列

在执行异步任务时，如dom事件，资源加载，setTimeout等，当异步任务完成时，如Ajax响应成功，setTimeout的时间到达规定时间时，这些异步任务的回调函数就会加入到消息队列中（FIFO）。

事件循环

当执行栈为空时，JavaScript引擎会开始依次执行消息队列中的回调函数，将其推入执行栈中。当该回调函数执行完毕后，执行栈又为空，JavaScript引擎继续执行消息队列的回调函数。

setTimeout

setTimeout表示在指定延迟后将回调函数加入消息队列中，在执行栈中的代码执行完成后，调用消息队列的消息回调函数。

如果setTimeout在1000ms后加入消息队列是队列中有消息，需要等待前面的消息执行完毕。

如果1000ms后加入消息队列中，但是此时执行栈中还有同步任务在执行，那么需要等待同步代码执行完毕，才执行setTimeout代码。

这两种情况都会导致setTimeout定时不准确。

传统回调函数的缺点

当多个请求相互依赖时，会产生回调的嵌套，而当这些请求的依赖顺序发生改变时，代码需要大规模的重构。

Promise的基本概念

Promise是JavaScript异步编程的一种解决方法，解决了回调嵌套的问题。Promise对象是对异步操作状态的描述，一个Promise有三种状态，pending，fullfilled，reject，这也是异步任务的三种状态。

fullfilled通过resolve函数设置，reject通过reject函数设置，如果没有调用以上方法，那么状态为pending。

当Promise对象的状态变为resolve或者reject后，会在下一次事件循环调用then回调函数。

有多个then回调函数时，会按顺序执行

Promise.resolve设置状态后，会再本轮事件循环的最后执行

then方法接收两个函数参数，第一个是resolve的回调函数，第二个是reject的回调函数，而catch方法其实是then方法省略了第一个参数的别名。

then方法默认返回一个新的Promise实例，且状态为resolve，默认是空值。如果返回一个值，则执行Promise.resolve(返回的值)。

commonJs 与 ES6

两个重要的区别
commonJs 是引入模块时是动态地按值拷贝，ES6 是静态分析时创建引用。
commonJs 模块会缓存值的结果，而 ES6 不会缓存。因为 cmj 是按值拷贝，类似于函数的传参，当值是引用类型时，改变该值时父模块的中引入的值也会跟着改变，而如果时原始类型，那么改变子模块的值父模块是没有反应的。而 ES6 中，不管是什么类型的值，只要改变了父模块的值也会跟着改变。

node 只能加载本地的 ES6 模块
如果文件不包含路径，则需要到 node_modules 下读取。
如果文件省略了后缀名，node 会依次尝试’./foo.mjs’,’./foo.js’,’./foo.json’,’./foo.node’，如果不存在
加载./foo/package.json 的 main 字段指定的脚本。如果依旧没有，加载./foo/index.mjs 等四个文件，如果没有则报错。

循环加载

CommonJS 模块无论加载多少次，都只会在第一次加载时运行一次，以后再加载，就返回第一次运行的结果，除非手动清除系统缓存。
commonJS 循环加载，加入 a 模块中引入了 b 模块，那么会先执行 b 模块，此时 b 模块中引入了 a 模块，a 模块不会执行，b 模块会只会引入 a 模块中已经执行的部分。等 b 模块执行完后，a 模块继续执行剩下的部分。
在 ES6 模块中，a 模块引入 b 模块，b 模块会先执行，b 模块中引入了 a 模块的接口，此时会默认这个接口是已经定义了的，继续执行 b 模块，如果使用了该接口且该接口在 a 模块的执行中还没定义，那么就会报错。

可以使用函数提升解决这个问题

作为父类的构造函数调用，this 指向子类实例。只能在子类的构造函数中调用，否则会报错。

作为对象使用时，在静态方法中指向父类，在普通方法中指向父类原型。

在子类的静态方法中通过 super 调用父类的方法时，方法内部的 this 指向当前的子类，而不是子类的实例。

可以在任意一个对象中，使用 super 关键字。

原生构造函数的继承

ES5 是先新建子类的实例对象 this，再将父类的属性添加到子类上，由于父类的内部属性无法获取，导致无法继承原生的构造函数。

ES6 允许继承原生构造函数定义子类，因为 ES6 是先新建父类的实例对象 this，然后再用子类的构造函数修饰 this，使得父类的所有行为都可以继承。

extends 关键字不仅可以用来继承类，还可以用来继承原生的构造函数。因此可以在原生数据结构的基础上，定义自己的数据结构。

NewObj 继承了 Object，但是无法通过 super 方法向父类 Object 传参。这是因为 ES6 改变了 Object 构造函数的行为，一旦发现 Object 方法不是通过 new Object()这种形式调用，ES6 规定 Object 构造函数会忽略参数。
