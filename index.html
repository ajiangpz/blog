CSS选择器

发表于 2019-05-26

权重计算

选择器权重

  • 行内样式 1000
  • id 选择器 100
  • class 选择器，属性选择器，伪类选择器 10
  • 元素选择器，伪元素选择器 1
  • 通配符选择器(*) 0

权重计算

选择器类型

通用选择器

  • id 选择器

  • class 选择器

    • 单独使用
    • 结合元素选择器
    • 多类选择器，选择同时包含且只包含这些类的元素

  • 元素选择器

属性选择器

  • [attr] 选择具有 attr 属性的元素
  • [attr=’value’] 选择 attr 属性为 value 的元素
  • [attr1][attr2] 选择同时具有 attr1 和 attr2 属性的元素

  • [attr~=’value’]选择属性中包含 value 的元素

  • [attr^=’value’] 选择 attr 属性值以 “value” 开头的所有元素

  • [attr$=’value’] 选择 attr 属性值以 “value” 结尾的所有元素

  • [attr*=’value’] 选择 attr 属性值中包含子串 “value”的所有元素

  • [attr|=’value’]选择 attr 属性值以 “value” 开头的元素，该值必须是整个单词

后代，子代，相邻选择器

  • div > p，选取 div 的子元素中的 p 元素
  • div p，选取 div 后代中的 p 元素
  • div+p，选取 div 后面相邻的 p 元素，没有相邻无法选中

伪类

  • :link
  • :visit
  • :hover
  • :active
  • :focus
  • :lang 为不同的语言定义特殊的规则。
  • :first-child

伪元素

  • :first-line
  • :first-letter
  • :before
  • :after

node基础

作用

  • 基于require模块化的JavaScript代码执行器

特点

  • 事件机制和异步IO

模块化

  • exports
    输出某个模块中的方法，变量

  • module
    重写整个输出的对象

  • require
    引入模块，可以是路径，内置模块

  • 主模块
    node命令行执行的函数

代码组织和部署

模块路径解析规则

  • 内置模块

  • node_modules目录

  • 自定义模块搜索路径
    在环境变量中配置node_path，加载模块时就会去该目录

包

  • 有多个依赖模块，将这些模块放在一个文件夹下，定义入口模块和输出模块，就形成了一个包
  • 包的入口模块默认时index.js,因此命名时如果为index.js就可以避免将输入模块写进路径中
  • 可以通过在包的根目录下创建package.json配置输入模块

命令行程序

  • window下配置命令行的路径，这样在任何地方都可以调用该程序

  • 在同一目录创建node-echo.cmd文件，在文件中加入以下代码，表示要执行的脚本

  • 在cmd规定的位置创建node-echo.js文件，编写要执行的内容
