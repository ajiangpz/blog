http协议

发表于 2019-07-02

http头部

  • 通用头部

    • cache-control
      • no-cache
        • 请求端包含该字段表明客户端不会接收缓存过的资源，缓存服务器必须把客户端的请求发送给源服务器(p87)
        • 响应端包含该指令表明缓存服务器不能对资源进行缓存，源服务器不再对缓存服务器中提出的资源有效性进行确认，且禁止其对响应进行缓存操作
      • no-store
        • 缓存不能再本地存储响应或者请求的任一部分
      • s-maxage=xxxx(单位:秒)
        • 只适用于供多个用户使用的公共缓存服务器（代理）,使用s-maxage后，maxage,expires无效
      • max-age=xxxx(单位:秒)
        • 客户端请求中包含该指令，如果判定缓存的时间比指定时间短，则接收缓存的资源，否则，缓存服务器需要将请求发送给源服务器
      • min-fresh=xxxx(单位:秒)
        • 返回缓存服务器中还未过指定时间的缓存资源
      • max-stale=xxxx(单位：秒)
        • 在指定的时间内都可以接收缓存资源，即使缓存资源已经过期
      • only-if-cached
        • 客户端只在缓存服务器本地缓存目标资源的情况下才会要求其返回，如过缓存中没有该资源，返回504状态码
      • must-revalidate
        • 代理必须向服务器再次验证该资源的有效性，如果代理无法连通服务器再次获取有效资源，缓存必须向客户端返回504状态码
      • proxy-revalidate
        • 所有的缓存服务器在接收到客户端带有该指令的请求返回响应之前，再次验证缓存的有效性。
      • no-transform
        • 无论是在请求还是在响应中，缓存都不能改变实体主体的媒体类型
    • connection
      • close
      • Keep-Alive 持久化连接
      • 不在转发给代理的首部字段
    • Date
      • Date: Tue, 03 Jul 2012 04:40:59 GMT
    • transfer-encoding
      规定了传输报文主体时采用的编码方式，HTTP/1.1 的传输编码方式仅对分块传输编码有效。
      • chunked 分块传输
    • upgrade
      • 首部字段 Upgrade 用于检测 HTTP 协议及其他协议是否可使用更高的版本进行通信，其参数值可以用来指定一个完全不同的通信协议。
    • Via
      • 使用首部字段 Via 是为了追踪客户端与服务器之间的请求和响应报文的传输路径。

  • 请求首部字段

    • accept : 通知服务器客户端可以处理的媒体类型及媒体相对优先级
      • 文本文件 text/html,text/plain,text/css
      • 图片文件 image/jpeg,image/gif,image/png
      • 视频文件 video/mpeg,video/quicktime
      • 二进制文件 application/octet-stream
    • accept-charset ：客户端支持的字符集及字符集的相对优先顺序
    • accept-encoding ：客户端支持的内容编码及内容编码的优先级顺序。
      • gzip：由文件压缩程序gzip生成的编码格式
      • compress：由UNIX文件压缩程序compress生成的编码格式
      • deflate：组合使用zlib格式及由deflate算法生成的编码格式
      • identity：不执行压缩或不会变化的默认编码格式
    • accept-language:zh-cn,zh;q=0.7
    • authorization：告知服务器用户代理的认证信息
    • expect： 告知服务器，期望出现的某种特定行为，目前只规定了100,continue
    • from： 告知服务器使用用户代理的用户的电子邮箱
    • Host：首部字段 Host 会告知服务器，请求的资源所处的互联网主机名和端口号。Host 首部字段在 HTTP/1.1 规范内是唯一一个必须被包含在请求内的首部字段。
    • If-Match:”123456” ：告知服务器匹配资源所用的实体标记（ETag）值
    • If-Modified-Since：If-Modified-Since 字段指定的日期时间后，资源发生了更新，服务器会接受请求，否则返回304
    • If-None-Match：字段值与Etag值不一致，可处理该请求，可用于获取最新的资源。
    • If-Range：告知服务器若指定的 If-Range 字段值（ETag 值或者时间）和请求资源的 ETag 值或时间相一致时，则作为范围请求处理。反之，则返回全体资源。
    • If-Unmodified-Since：如果在指定的时间之后服务器的资源未发生更新，则处理请求，否则，以状态码412作为响应返回。
    • Max-forwards：指定转发的最多次数，每转发一次，该值减1。为0时服务器直接返回响应。
    • Range：bytes=5001-10000，告知服务器资源的范围，服务器接收到该字段的请求会返回206响应，无法处理范围请求时，返回200 OK
    • Referer：告知服务器请求的原始资源的 URI
    • User-Agent：User-Agent 会将创建请求的浏览器和用户代理名称等信息传达给服务器。
  • 响应首部字段
    • Age：服务器创建响应到现在的市场，如果是缓存服务器，则返回缓存服务器再次验证资源的有效性到现在的时间
    • Accept-Ranges
      • none：不能处理范围请求
      • bytes：可以处理范围请求
    • ETag：将资源以字符串形式做唯一性标识的方式，服务器会为每一份资源分配对应的ETag，有强ETag和弱ETag之分
    • Location：配合3xx;edirection的响应，提供重定向的URI
    • Server：告知客户端当前服务器上安装的Http服务器应用程序的信息
    • Vary：源服务器控制缓存服务器只返回和该字段的值一致的缓存，否则，须先从源服务器获取资源后才能作为响应返回。
  • 实体首部字段
    • Allow：告知客户端服务器支持的Http方法，服务器接收到不支持的Http方法时，返回405及支持的Http方法
    • Content-Encoding：告知客户端实体的内容编码方式，即压缩方式
      • gzip
      • compress
      • deflate
      • identify
    • Content-Language：告知客户端实体主体使用的自然语言（中文或者英文）
    • Content-Length：表明了实体主体部分的大小
    • Content-Location：Content-Location 表示的是报文主体返回资源对应的 URI。
    • Content-Range: bytes 5001-10000/10000，表示当前发送部分和整个实体大小
    • Content-Type: text/html; charset=UTF-8，实体主体内对象的媒体类型。
    • Expires：将资源失效的日期告诉的客户端。当超过指定时间后，缓存在请求发送过来时会向服务器请求资源，否则，返回缓存。
    • Last-Modified：资源最终修改的时间。
  • 为Cookie服务的首部字段
    • Set-Cookie
      • NAME=VALUE
      • expires=DATE，省略时浏览器会话结束cookie失效。一旦Cookie被发送至客户端，服务器不存在显示删除Cookie的方法。可以通过覆盖以过期的Cookie，实现实质的删除操作
      • path=PATH，限制Cookie的发送范围的文件目录
      • domain=域名，以domain指定的域名结尾的域名都可以发送Cookie，不指定默认为创建Cookie的服务器的域名
      • Secure，只在Https发送Cookie
      • HttpOnly，不允许JavaScript操作Cookie
    • Cookie：status=enable

rem是什么

1rem=根元素fontsize的值,因此设置rem的值时，可以考虑两种情况，一种是将根元素的fontsize设置成整数，比如100px，50px，一般不推荐设置的值小于12px,因为chrome浏览器会将有最小字体的限制。另一种方案是将rem设置成设计稿的整数倍，比如1rem=64px,总之，设置rem应尽量让数值取整。

层叠上下文

发表于 2019-06-07

层叠上下文

层叠上下文的概念

  • 抽象的概念，在层叠上下文中，元素会按照一定的规则和顺序进行排列覆盖。不同的层叠上下文元素的子元素之间不会相互影响，他们的层叠等级由父元素决定。

    层叠上下文的产生

  • 根元素
  • 定位元素且z-index为数值
  • css3，如transform,子元素的z-index不为auto的flex布局

    层叠上下文的特点

  • 后来居上,同一层叠水平的元素，根据元素在dom中的先后顺序进行排列。
  • 根据z-index或层叠顺序排列

层叠顺序

层叠顺序表

  • background/border
  • 负z-index
  • block块状元素
  • float浮动盒子
  • inline/inline-block盒子
  • z-index为0或者auto，不依赖z-index的层叠上下文
  • 正z-index
  • 层叠顺序演示

    特殊情况

  • 元素一旦定位，其z-index自动生效默认值为auto，等价于0
    与不依赖z-index的层叠上下文（如css3）层叠等级相同，遵循后来居上的原则。

flex布局

发表于 2019-06-02

属性

父元素属性

  • flex-direction 规定主轴方向，子元素的流动方向
    • column 规定y轴为主轴，子元素自上而下流动
    • row 规定x轴为主轴，子元素从左到右流
    • column-reverse 规定y轴为主轴，子元素自下而上流动
    • row-reverse 规定x轴为主轴，子元素从右到左流动
  • flex-wrap 规定子元素换不换行
    • wrap 换行
    • nowrap 不换行,元素按比例缩小
    • wrap-reverse 向上换行
  • flex-flow flex-direction和flex-wrap的简写
  • justify-content 规定主轴上元素的排列方式
    • center
    • flex-end
    • flex-start
    • space-around
    • space-between
  • align-items 规定子元素的对齐方式
    • center 居中
    • flex-end 下对齐
    • flex-start 上对齐
    • strech 拉伸
  • align-content 规定交叉轴的多行元素排列

    子元素属性

  • flex-grow 拉伸倍数
  • flex-shrink 缩小倍数
  • flex-basic 拉伸或缩小的基值,会覆盖width的值
    • 设置为0，宽度也是元素内容，input有最小宽度值，在火狐下141px，不管有没有内容
    • 设置为auto时，即为元素的width值
    • 设置为百分比时会根据父元素的宽度计算宽度
  • flex 上述三个值的按顺序简写
    • auto=> 1 1 auto
    • 1=>1 1 0%
    • 10px=>1 1 10px
    • 40% =>1 1 40%
    • 1 10px =>1 1 10px
    • none => 0 0 auto
    • 默认=>0 1 auto
  • 缩放计算 所有子元素的flex-basic相加,剩余部分按flex-grow或者flex-shrink分配

权重计算

选择器权重

  • 行内样式 1000
  • id 选择器 100
  • class 选择器，属性选择器，伪类选择器 10
  • 元素选择器，伪元素选择器 1
  • 通配符选择器(*) 0

权重计算

选择器类型

通用选择器

  • id 选择器

  • class 选择器

    • 单独使用
    • 结合元素选择器
    • 多类选择器，选择同时包含且只包含这些类的元素

  • 元素选择器

属性选择器

  • [attr] 选择具有 attr 属性的元素
  • [attr=’value’] 选择 attr 属性为 value 的元素
  • [attr1][attr2] 选择同时具有 attr1 和 attr2 属性的元素

  • [attr~=’value’]选择属性中包含 value 的元素

  • [attr^=’value’] 选择 attr 属性值以 “value” 开头的所有元素

  • [attr$=’value’] 选择 attr 属性值以 “value” 结尾的所有元素

  • [attr*=’value’] 选择 attr 属性值中包含子串 “value”的所有元素

  • [attr|=’value’]选择 attr 属性值以 “value” 开头的元素，该值必须是整个单词

后代，子代，相邻选择器

  • div > p，选取 div 的子元素中的 p 元素
  • div p，选取 div 后代中的 p 元素
  • div+p，选取 div 后面相邻的 p 元素，没有相邻无法选中

  • :active
  • :focus
  • :lang 为不同的语言定义特殊的规则。
  • :first-child
