Object.assign()

Object.assign()用于对象可枚举属性的合并，第一个参数是目标对象，其他参数都是源对象，出现相同属性时后面的对象会覆盖前面的对象。

如果参数不是对象，该方法会将其转为为对象，然后返回，但是原来的值没有变化。

如果无法转化为对象，则报错。

如果非对象参数出现在源对象的位置（即非首参数），那么处理规则有所不同。首先，这些参数都会转成对象，如果无法转成对象，就会跳过。这意味着，如果 undefined 和 null 不在首参数，就不会报错。
其他类型的值（即数值、字符串和布尔值）不在首参数，也不会报错。但是，除了字符串会以数组形式，拷贝入目标对象，其他值都不会产生效果。

属性的可枚举性和遍历

可枚举的属性即 enumerator 属性为 true 的属性，这些属性可以被 for..in..，object,keys()，Object.assign()，JSON.stringify，等方法遍历，其中 for…in…会遍历原型链的属性。引入可枚举的目的是为了让某些属性可以规避掉 for…in..操作，否则很多对象原型上的方法如 toString()都会被遍历出来。

ES6 规定，所有 Class 的原型的方法都是不可枚举的。

操作中引入继承的属性会让问题复杂化，大多数时候，我们只关心对象自身的属性。所以，尽量不要用 for…in 循环，而用 Object.keys()代替。

ES6 遍历属性的 5 种方法

  1. for…in…
    遍历对象自身和原型链的可枚举属性，不含 symbol
  2. Object.keys()
    遍历对象自身的可枚举属性，不含 symbol
  3. Object.getOwnPropertyNames()
    遍历对象自身可枚举和不可枚举属性，不包含 symbol
  4. Object.getOwnPropertySymbols
    遍历对象自身所有 Symbol 属性
  5. Reflect.ownKeys(obj)
    Reflect.ownKeys 返回一个数组，包含对象自身的所有键名，不管键名是 Symbol 或字符串，也不管是否可枚举。

es6 对象中的 super 指向对象的原型

注意，super 关键字表示原型对象时，只能用在对象的方法之中，用在其他地方都会报错。

copyWithin()

copyWithin(target,start,end)，将指定位置即 target 的值替换为 start 到 end 之间的值，不包括 end。
target（必需）：从该位置开始替换数据。如果为负值，表示倒数。

find 方法的回调函数可以接受三个参数，依次为当前的值、当前的位置和原数组。

另外，这两个方法都可以发现 NaN，弥补了数组的 indexOf 方法的不足。

参数默认值

ES6 允许为函数的参数设置默认值，即直接写在参数定义的后面。

参数变量是默认声明的，所以不能用 let 或 const 再次声明。

使用参数默认值时，函数不能有同名参数。

参数默认值不是传值的，而是每次都重新计算默认值表达式的值。也就是说，参数默认值是惰性求值的。

与解构赋值默认值结合使用

参数默认值的位置

通常情况下，定义了默认值的参数，应该是函数的尾参数。如果非尾部的参数设置默认值，实际上这个参数是没法省略的。

函数的 length 属性

指定了默认值以后，函数的 length 属性，将返回没有指定默认值的参数个数。也就是说，指定了默认值后，length 属性将失真。length 属性的含义是，该函数预期传入的参数个数。某个参数指定默认值以后，预期传入的参数个数就不包括这个参数了。

如果设置了默认值的参数不是尾参数，那么 length 属性也不再计入后面的参数了。

箭头函数没有参数或者有多个参数时，需要用圆括号代替参数或者将多个参数包围起来，如果只有一个参数可以省略圆括号。

箭头函数与变量解构结合使用

使用箭头函数的注意

  1. 箭头函数的 this 对象就是定义时所在的对象。
  2. 不可以当构造函数使用，即不可以使用 new 命令。
  3. 不能使用 arguments 对象，可以用 rest 参数代替。
  4. 不可以使用 yield 命名，不能用作 generator 函数。

箭头函数中没有 this，arguments，super，这些对象都是从词法作用域中获取的。因此用 call，bind，apply 等方法是无法改变箭头函数中的 this 指向的。
稳定的 this 指向，使箭头函数非常适合用来封装回调函数。

当点击一个按钮时，普通函数回调函数的 this 一般会指向 document 对象，这样就容易出现 this 对象上没有相应的方法，箭头函数解决了这个问题。

Symbol 的定义

Symbol 是 JavaScript 第六种基本类型，代表一个独一无二的值。

Symbol 的创建

由于 Symbol 是基本类型，因此不能用 new Symbol()来创建，否则会报错。
创建一个无标记的 Symbol

创建一个有标记的 Symbol，主要目的方便区分不同的 Symbol，否则打印出来都是 Symbol();

通过 Symbol.for(key)，可以创建一个类似于单例模式的 Symbol，以后每一次调用 Symbol.for(key)都是返回同一个对象。

需要注意的是，Symbol.for 为 Symbol 值登记的名字，是全局环境的，可以在不同的 iframe 或 service worker 中取到同一个值。
通过调用 Symbol.keyfor(symbol 值)，可以得到该 Symbol 值创建时对应的 key。

Symbol 作为属性值

在对象的内部，使用 Symbol 值定义属性时，Symbol 值必须放在方括号之中。

Symbol 属性值的遍历

Symbol 作为属性名，该属性不会出现在 for...in、for...of循环中，也不会被Object.keys()、Object.getOwnPropertyNames()、JSON.stringify()返回。
但是，它也不是私有属性，有一个 Object.getOwnPropertySymbols 方法，可以获取指定对象的所有 Symbol 属性名。
Object.getOwnPropertySymbols 方法返回一个数组，成员是当前对象的所有用作属性名的 Symbol 值。

let-and-const

发表于 2019-10-04

let不存在变量提升，同时let还会绑定所在的代码块，即大括号包含的区域。在这个代码块内如果在let声明之前使用变量，就会报ReferenceError错误。

let 不可以重复声明变量。

let块级作用域和函数声明
在 ES6 中可以在块级作用域中声明函数，声明行为类似于let。但是在实际 ES6 浏览器中，函数声明有自己的方式。
允许在块级作用域声明函数，并且函数会提升到该块级作用域顶部，同时会提升到全局作用域或者函数作用域顶部
。

由于不同环境下造成的差异较大，因此在块级作用域中应该避免声明函数，或者应该使用函数表达式。

const声明变量的行为与let基本一致，const表示声明一个常量，不同多次赋值，因此一定要在声明的时候同时初始化。

当const声明的常量是一个对象时，实际上不变性指的只是对象的地址值保持不变。因此堆中的对象的属性值发生改变时，const是不会报错的。

但是当常量的地址值改变了，此时常量的值就真正改变了，就会报错。

要包持对象的不变性，可以对对象进行封装或者冻结。

ES6 中一共有六种声明变量的方法：var，function，let，const，class，import

顶层对象的属性
var，function声明的全局变量依旧是顶级对象的属性，let，const，class命令声明的全局变量不属于顶级对象。

原型

发表于 2019-09-28

object.create会创建一个对象，并把这个对象的[prototype]和指定的对象关联。

属性设置和屏蔽
给一个对象设置属性时，当一个对象的原型链上已经存在该属性，此时会发生三种情况

  • 原型链上的属性是可写的(writable:true)，那么 js 会在对象上创建属性。

  • 原型链上的属性是只读的，那么 js 不会在对象上创建属性，如果在严格模式下，还会报错。

  • 原型链上的属性存在 setter，那么 js 会调用这个 setter，不会将在对象上创建属性。
