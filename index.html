commonJs 与 ES6

两个重要的区别
commonJs 是引入模块时是动态地按值拷贝，ES6 是静态分析时创建引用。
commonJs 模块会缓存值的结果，而 ES6 不会缓存。因为 cmj 是按值拷贝，类似于函数的传参，当值是引用类型时，改变该值时父模块的中引入的值也会跟着改变，而如果时原始类型，那么改变子模块的值父模块是没有反应的。而 ES6 中，不管是什么类型的值，只要改变了父模块的值也会跟着改变。

循环加载

CommonJS 模块无论加载多少次，都只会在第一次加载时运行一次，以后再加载，就返回第一次运行的结果，除非手动清除系统缓存。
commonJS 循环加载，加入 a 模块中引入了 b 模块，那么会先执行 b 模块，此时 b 模块中引入了 a 模块，a 模块不会执行，b 模块会只会引入 a 模块中已经执行的部分。等 b 模块执行完后，a 模块继续执行剩下的部分。
在 ES6 模块中，a 模块引入 b 模块，b 模块会先执行，b 模块中引入了 a 模块的接口，此时会默认这个接口是已经定义了的，继续执行 b 模块，如果使用了该接口且该接口在 a 模块的执行中还没定义，那么就会报错。

可以使用函数提升解决这个问题

基本用法

promise是一个构造函数，参数是一个函数，且这个函数接收两个函数参数，一个是 resolve 函数，一个是 reject 函数。resolve 函数负责将 promise 的状态设置为 fullfill 即完成状态，而 reject 负责将 promise 状态设置为 reject 的状态，即失败状态。promise 的实例对象有一个 then 方法和 catch 方法，在 fullfill 状态调用 then 回调函数，在 reject 状态调用 catch 函数。

如果调用 resolve 函数和 reject 函数时带有参数，那么它们的参数会被传递给回调函数。reject 函数的参数通常是 Error 对象的实例，表示抛出的错误；resolve 函数的参数除了正常的值以外，还可能是另一个 Promise 实例，比如像下面这样。

p2 的状态由 p1 决定，因此 p2 的 then，catch 都是针对 p1 的。

Promise.prototype.then

promise原型对象上的 then 方法是指定状态之后执行的回调函数，then 方法两个函数参数，第一个是 resolve 对应的回调函数，第二个是 reject 对应的回调函数。then 方法返回一个新的 promise 实例，因此可以链式调用。

在链式调用中返回 promise

Promise.prototype.catch

catch 方法用来捕获 promise 的错误，即 reject 状态。当 promise 中的抛出一个错误时，也会触发 catch。

catch 相当于 then(null,catch)的缩写
当有多个 promise 链式调用时，应该将 catch 放到最后，捕获所有 promise 的错误。

如果没有使用 catch 方法指定错误处理的回调函数，Promise 对象抛出的错误不会传递到外层代码，即不会有任何反应。

Promise.prototype.finally

finally用于指定不管promise最后的状态是什么都会执行的操作，finally(callback)回调函数不接收任何参数，可以用()=>dosomething 代替，也就是说 finally 的执行与否与状态无关。

Promise.all

all方法接收一个 promise 实例组成的数组，或者具有迭代器的结构，返回一个新的promise实例。当数组中所有的promise为 resolve 后，返回的promise执行的状态为resolve返回所有resolve组成的数组，当数组中有一个状态为reject，返回的promise的状态为reject，且为第一个返回的reject。

有一个reject就触发catch。

如果数组中的promise中有catch回调函数，那么该promise返回的是catch返回的promise，而且其状态为resolve，返回为空值。

node 环境下数组中的promise和all函数返回的promise都会触发catch函数。

proxy 是什么？

proxy 是一个构造函数，用来构造指定对象的代理对象。

基本用法

Proxy 函数接受两个参数，第一个是要代理的对象，第二个是配置对象，其中包括要被代理的操作，比如 get 操作（获取属性），set 操作（设置属性），apply（函数被调用）等操作。这些操作执行时，就会被拦截执行另外的操作。

可以将 proxy 设置为目标对象的一个属性，这样就可以通过对象属性访问代理对象。

proxy 也可以作为其他对象的原型

proxy 拦截方法

get()

接收三个参数，第一个为目标对象，第二个为属性，第三个为 proxy 实例
使用 get 拦截，实现数组读取负数的索引。

利用 get 参数可以实现函数的链式调用
思路：先将访问的属性如 double，pow 等存放在一个数组中，且每次访问都返回 proxy 对象。当调用 get 方法时，一次性返回或者如果属性是函数则执行所有函数。

set()

set 在代理对象设置属性时调用，接收四个参数，第一个是目标对象，第二个是属性名，第三个是属性值，第四个是 proxy 实例。

对年龄的设置限制条件，超过 200 则报错。
set 和 get 还可以结合使用用来设置私有属性

Apply

apply 拦截函数的调用，call，apply 等操作。

has

拦截 HasProperty，即判断对象是否有某个属性时，如 in 操作。可以利用这个方法隐藏私有属性。

has 拦截只对 in 运算符生效，对 for…in 循环不生效，导致不符合要求的属性没有被 for…in 循环所排除。

deleteProperty

deleteProperty(target, propKey)：拦截 delete proxy[propKey]的操作，返回一个布尔值。

ownkeys

ownKeys(target)：拦截 Object.getOwnPropertyNames(proxy)、Object.getOwnPropertySymbols(proxy)、Object.keys(proxy)、for…in 循环，返回一个数组。该方法返回目标对象所有自身的属性的属性名，而 Object.keys()的返回结果仅包括目标对象自身的可遍历属性。

getOwnPropertyDescriptor(target, propKey)

拦截 Object.getOwnPropertyDescriptor(proxy, propKey)，返回属性的描述对象。

defineProperty(target, propKey, propDesc)

拦截 Object.defineProperty(proxy, propKey, propDesc）、Object.defineProperties(proxy, propDescs)，返回一个布尔值。

preventExtensions

拦截 Object.preventExtensions(proxy)，返回一个布尔值。

getPrototypeOf(target)

拦截 Object.getPrototypeOf(proxy)，返回一个对象。

isExtensible(target)

拦截 Object.isExtensible(proxy)，返回一个布尔值。

setPrototypeOf(target, proto)

拦截 Object.setPrototypeOf(proxy, proto)，返回一个布尔值。如果目标对象是函数，那么还有两种额外操作可以拦截。

construct(target, args)

拦截 Proxy 实例作为构造函数调用的操作，比如 new proxy(…args)。

解决办法是可以返回一个 this 与目标对象强绑定的函数

成员只能是对象，且不能被遍历。weakSet 结构中对对象都是弱引用，也就是当 weakSet 中的对象的其他引用为 0，这些对象可以被回收。

Map 结构

Js 对象本质上是键值对的集合（Hash 结构），但是传统上只能用字符串当作键。如将 dom 节点作为键值则 dom 节点会被转化为字符串[object HTMLDivElement]
Map 结构可以将任何类型的数据作为键值，因为 Map 结构不会管键值到底是什么，只会找到键值所在的内存地址。

Map 的键实际上是跟内存地址绑定的，只要内存地址不一样，就视为两个键。这就解决了同名属性碰撞（clash）的问题，我们扩展别人的库的时候，如果使用对象作为键名，就不用担心自己的属性与原作者的属性同名。

Map 实例的方法
Map.prototype.set()返回 Map 实例，可以作链式操作，Map.prototype.get(k)，获取键值对应的键名，Map.prototype.has(key)，Map.prototype.delete(key)这些方法都和 Set 一致。

遍历方法包括 keys(),values(),entries(),forEach(),但是默认是可以被 for…of…遍历的，默认的遍历器对象为 entries()返回的对象。

Map 可以转化为数组进行处理后再转化为 Map 结构，也就是说可以用数组的方法对象 Map 数据进行处理

Object.assign()

Object.assign()用于对象可枚举属性的合并，第一个参数是目标对象，其他参数都是源对象，出现相同属性时后面的对象会覆盖前面的对象。

如果参数不是对象，该方法会将其转为为对象，然后返回，但是原来的值没有变化。

如果无法转化为对象，则报错。

如果非对象参数出现在源对象的位置（即非首参数），那么处理规则有所不同。首先，这些参数都会转成对象，如果无法转成对象，就会跳过。这意味着，如果 undefined 和 null 不在首参数，就不会报错。
其他类型的值（即数值、字符串和布尔值）不在首参数，也不会报错。但是，除了字符串会以数组形式，拷贝入目标对象，其他值都不会产生效果。

属性的可枚举性和遍历

可枚举的属性即 enumerator 属性为 true 的属性，这些属性可以被 for..in..，object,keys()，Object.assign()，JSON.stringify，等方法遍历，其中 for…in…会遍历原型链的属性。引入可枚举的目的是为了让某些属性可以规避掉 for…in..操作，否则很多对象原型上的方法如 toString()都会被遍历出来。

ES6 规定，所有 Class 的原型的方法都是不可枚举的。

操作中引入继承的属性会让问题复杂化，大多数时候，我们只关心对象自身的属性。所以，尽量不要用 for…in 循环，而用 Object.keys()代替。

ES6 遍历属性的 5 种方法

  1. for…in…
    遍历对象自身和原型链的可枚举属性，不含 symbol
  2. Object.keys()
    遍历对象自身的可枚举属性，不含 symbol
  3. Object.getOwnPropertyNames()
    遍历对象自身可枚举和不可枚举属性，不包含 symbol
  4. Object.getOwnPropertySymbols
    遍历对象自身所有 Symbol 属性
  5. Reflect.ownKeys(obj)
    Reflect.ownKeys 返回一个数组，包含对象自身的所有键名，不管键名是 Symbol 或字符串，也不管是否可枚举。

es6 对象中的 super 指向对象的原型

注意，super 关键字表示原型对象时，只能用在对象的方法之中，用在其他地方都会报错。
目前，只有对象方法的简写法可以让 JavaScript 引擎确认，定义的是对象的方法。

对象的解构赋值将对象自身的，可遍历的属性分配到指定的对象上。
